# ADR-0007：假说与原文的语义关系由假设生成 Agent 标注，一假说一关系

## 状态

已接受（2026-07-10）

## 背景

回写操作（替换/改写/补充）的唯一分派依据是「假说与原文的语义关系」（对立/递进/扩展，见 ADR-0006）。
但该关系在 PRD 中无归属：不知由谁分类、在哪步产出、存哪个字段。`candidate_hypotheses` 原结构 `{ text, confidence }` 也无 relation 字段。
且一条假说可能同时踩多种关系（如既改数字又补归因）。

## 决策

1. **关系归属假设生成 Agent**：生成假说的同刻同步标注关系，因为假说意图在诞生时最清楚，事后反推是丢信息后的猜测。
2. **落入数据结构**：
   ```
   candidate_hypotheses: { text, confidence, relation: 'oppose' | 'advance' | 'expand' }
   ```
   - `oppose`（对立）→ 替换
   - `advance`（递进）→ 改写
   - `expand`（扩展）→ 补充
3. **一假说一关系（单一意图约束）**：一条假说只允许一种关系。若需同时「改数字 + 补归因」，必须拆成两条假说（一条 `oppose`、一条 `expand`）。

## 权衡

- 拆条增加假说数量，但换取回写通道唯一、不打架，且 HITL-2 支持分条勾选。
- 选「生成时标注」而非「独立分类 Agent」：避免生成与分类解耦后的信息损耗与额外一跳。

## 影响

- `candidate_hypotheses` 新增 `relation` 字段。
- HITL-2 用户可对同一节点的多条假说分条采纳/驳回。

## 备注：候选数量

User Story 提到「3 个候选修订假设」——`3` 仅为 UI 典型呈现期望，**非硬约束**。拆条后假说数量天然可变（0..N）。
