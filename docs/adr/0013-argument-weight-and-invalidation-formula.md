# ADR-0013：权重由解析 Agent 按 rubric 赋值，失效用剩余支撑率公式判定

## 状态

已接受（2026-07-10）。起步方案，参数后期回归调优。

## 背景

影响传导按 `argument_weight` 计算「剩余支撑力度」判 `invalid`，但 PRD 未定义权重来源，也未给失效判定公式，导致 `invalid` 判定不可信。

## 决策

1. **权重赋值**：解析 Agent 建树时给每个节点打 `argument_weight` (0-100)，依**明文 rubric**（带数据/引源的直接论据高分、泛泛断言低分），而非纯拍脑袋。
2. **失效判定公式**（影响传导用）：
   - 父论点 **剩余支撑率 = Σ(存活直接子节点权重) / Σ(全部直接子节点权重)**。
   - `< 0.5` → 父节点 `invalid`；
   - `0.5 ~ 0.7` → 贴「弱化」批注，不失效；
   - `≥ 0.7` → 不受影响。
3. **阈值与 rubric 均为可调参数**，后期用真实文档回归调优。

## 权衡

- 先用确定性公式而非 LLM 自由判断，保证 `invalid` 可解释、可测试、可调。
- 起步阈值（0.5 / 0.7）是经验值，未经数据验证，明确标注为待调优。

## 影响

- 权重来源明确（解析 Agent + rubric），非黑盒。
- 失效判定可单测（给定子节点权重与存活情况，断言父节点是否 invalid）。
