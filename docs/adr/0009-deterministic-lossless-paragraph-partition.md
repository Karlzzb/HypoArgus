# ADR-0009：段落切分为确定性无损纯代码步骤，独立于 LLM 解析

## 状态

已接受（2026-07-10）

## 背景

字节级还原承诺 = 未变更段落按 `paragraph_id` 从只读原文表原样拷回。
但论证树解析由 LLM 完成，天然模糊（可能漏句、合段、误判标题、忽略空行）。
若只读原文表从 LLM 解析结果反推，则 LLM 一抖动，字节级承诺当场崩盘。

## 决策

将「段落切分」与「论证树解析」彻底解耦为两个独立步骤：

1. **第一步——确定性无损切分（纯代码，零 LLM）**：
   - 用纯规则按 **Markdown 块级元素 + 空行边界**切分（一个标题/段落/列表项/代码块各成一段）。
   - 直接灌入只读原文段落表，并校验**分区不变式**：`assert(所有段落按序拼接 == 原始输入)`（含空行、缩进、换行、末尾空格）。
   - 字节级还原由此成为**代码级确定**，与 LLM 质量无关。

2. **第二步——LLM 解析论证树（只引用、不搬运）**：
   - 输入是已切好、带 `paragraph_id` 的段落序列。
   - LLM 只识别每段内的论证节点、挂树、回填 `paragraph_id` 指针。
   - **无权新建、改写、重排段落**。解析再烂也只影响树结构，动不了字节底座。

## 权衡

- 纯代码切分无法「智能」合并语义连续的跨段内容，但换取字节承诺的确定性，值得。

## 影响

- 只读原文段落表在 LLM 介入前就已固化并通过分区不变式校验。
- Testing Decisions 的「字节级对齐」可兑现——它不经过 LLM。
- 切分粒度：Markdown 块级 + 空行边界。
