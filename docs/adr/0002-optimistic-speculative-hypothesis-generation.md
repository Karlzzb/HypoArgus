# ADR-0002：假设生成采用全量投机 + 节点类型白名单

## 状态

已接受（2026-07-10）

## 背景

假设生成 Agent 与事实验证 Agent「乐观并行」同时启动。
但 PRD 又将假设生成的对象描述为「针对存疑/缺乏支撑的节点」——而在验证出结果前，无从得知哪些节点会被判为 `doubtful/error`。
这是因果悖论：并行启动时，验证结论尚不存在。

## 决策

采用**全量投机生成（解法 A）**，而非「先预筛再生成」：

1. 假设生成**不依赖验证结论**，在验证并行进行的同时，对目标节点无差别投机生成候选假设。
2. 由**双轨合并算子**在两轨都完成后裁剪：`credible` 节点的假设被 Drop，`doubtful/error` 节点的假设被激活。
3. **成本闸门**：投机生成只覆盖 `evidence` 与 `sub_claim` 两类节点。跳过 `main_claim`（靠影响传导而非直接替换）、`qualification` 与影子节点。

## 权衡

- 选 A 而非 B（轻量预筛）：换取真并行、逻辑自洽、最贴合「乐观并行压榨吞吐」的原始意图。
- 代价：对最终会被判 `credible` 的节点也生成了假设，这部分算力被 Merge 裁剪浪费。用节点类型白名单把生成量收敛到可接受范围。

## 影响

- 生成量上限 ≈ (`evidence` 数 + `sub_claim` 数)。
- Merge 算子是唯一的裁剪点，必须在两轨都完成后运行。
