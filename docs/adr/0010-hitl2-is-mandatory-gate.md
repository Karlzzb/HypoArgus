# ADR-0010：HITL-2 为不可跳过的硬闸门，系统绝不自动改原文

## 状态

已接受（2026-07-10）

## 背景

HITL-1（结构确认）PRD 明说可跳过；但 HITL-2（修订内容确认）未提是否可跳过，而任务配置又有「是否启用人工确认」开关，留下危险模糊地带。
若 HITL-2 可跳过并自动采纳所有 supported 假说，则系统在无人拍板下自动替换/改写专业文档论据——正是 PRD Problem 4「AI 全量重写失控」的变种，违背系统「保护原文、精准可控」立身之本。

## 决策

1. **HITL-2 恒为硬闸门，不可跳过，绝不自动采纳假说。**
2. 任务配置「是否启用人工确认」开关**只作用于 HITL-1**；HITL-2 无法关闭。
3. **空过口径**：若某次运行所有节点均 `credible`、无任何 supported 假说、无 error/doubtful，则 HITL-2 呈现「无需修订」一键通过——这是「闸门内无待办」，不是跳过闸门。

## 权衡

- 放弃「全自动无人值守跑批」场景，换取「原文改写必经人判」的安全底线。若未来确需无人值守，须另立显式高风险模式，不得复用此开关。

## 影响

- HITL-1 可跳过（最坏只是树切分不完美，不动原文）。
- HITL-2 是唯一决定「原文是否被改」的关卡，恒定开启。
