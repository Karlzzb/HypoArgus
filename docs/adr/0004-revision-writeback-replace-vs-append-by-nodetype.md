# ADR-0004：回写按 node_type 分流——论据替换、论点追加

## 状态

~~已接受（2026-07-10）~~

## 背景

PRD 2.6 并列了两个互斥的回写动作：「对确认段落局部重写」（替换）与「新假设追加在段落末尾」（追加）。
二者产品形态完全不同：替换产出干净新文，追加保留原文 + 勘误式脚注。需要消歧。

## 决策

不做一刀切，按被采纳假设所挂载节点的 `node_type` 分流：

- **`evidence`（论据）节点采纳的假设 → 替换（rewrite）**。事实错误应改正，原句进重写通道。
- **`sub_claim` / `main_claim`（论点）层的假设，及「补充论据」等增量 → 段尾追加（append）**，带专属修订标识，保留原论点以供审计。

即：**修订 = 事实层替换 + 论点层追加**。

## 权衡

- 相比「一律追加」（对原文零破坏但啰嗦）与「一律替换」（干净但丢审计痕迹），分流兼顾了正确性与可审计性。
- 代价：回写逻辑需读取节点类型分派通道，略增复杂度。

## 影响

- 字节级测试：替换段与追加段都算「已变更」，不承诺字节对齐；仅完全未触及的段落承诺字节无损。
- 追加内容须带「专属修订标识」，便于合规审计一眼识别新注入内容。
