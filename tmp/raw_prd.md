# 论证驱动型文档修订多智能体系统 PRD（纯智能体协作·修正版）
**版本**：V1.1  
**设计范围**：仅包含多智能体架构、角色定义、编排逻辑、检索体系、HITL节点与协作协议，不含前端交互与界面层内容  
**核心约束**：以论证树为唯一数据主干，无知识三元组设计，检索能力公共复用，人在回路关键节点可控

---

## 1. 系统概述
### 1.1 系统定位
一套面向专业文档修订场景的后端多智能体协作系统，以**标准化论证树为唯一数据主干**，通过「主干解析-双任务执行-公共检索支撑-两级门禁-人在回路确认」的分层编排，完成文档论证结构解析、可验证假设生成、多源事实验证、逻辑影响传导与合规修订输出全链路自动化处理。系统采用可插拔设计，支持全量修订、单任务调用两种运行模式。

### 1.2 设计目标
1.  **结构统一**：全链路共享同一套论证树主干，避免多模块重复解析、语义口径不一致。
2.  **能力解耦**：检索能力公共复用，任务能力独立封装，每个智能体仅承担单一原子职责，可独立迭代、按需装配。
3.  **闭环验证**：所有事实论断与生成假设均通过多源检索完成支撑/反驳校验，无依据不输出结论。
4.  **人机可控**：关键节点嵌入人工确认环节，人工干预结果直接同步至论证树，保障最终输出符合预期。

### 1.3 能力边界
- 覆盖能力：论证结构解析、可验证假设生成、多源信息检索、论据事实验证、逻辑影响传导、全文一致性校验、修订内容回写
- 不覆盖：通用文本润色、格式排版处理、多模态内容解析、前端交互界面渲染

## 2. 整体智能体架构
采用「1中枢 + 1主干 + 双任务 + 1检索层 + 2HITL节点 + 1门禁1输出」的分层编排拓扑，所有智能体通过全局调度Agent转发数据，无跨模块直接调用。

```
                    └─────────┬───────────┘
                              │
          ┌───────────────────┼───────────────────┐
          │                   │                   │
┌─────────▼─────────┐  ┌──────▼──────┐  ┌────────▼─────────┐
│ 论证结构解析Agent │  │  HITL节点组  │  │ 一致性校验Agent   │
│ （唯一主干）      │  │ 1.结构确认  │  │ （质量门禁）      │
│                   │  │ 2.内容确认  │  │                   │
└─────────┬─────────┘  └─────────────┘  └─────────┬─────────┘
          │                                       │
          └───────────────────┬───────────────────┘
                              │
                    ┌─────────▼───────────┐
                    │  双任务执行层       │
                    │  假设生成Agent      │
                    │  事实验证Agent      │
                    │  影响传导Agent      │
                    └─────────┬─────────┘
                              │
                    ┌─────────▼───────────┐
                    │  公共检索能力层     │  ← 底层公共支撑，双任务统一调用
                    │ 网络检索Agent       │

                    │ 结构化数据检索Agent │

                    ┌─────────▼───────────┐
                    │  修订回写Agent      │  ← 结构化结果转自然语言输出
                    └─────────────────────┘
```

## 3. 智能体角色详细定义
### 3.1 中枢层：全局调度Agent
- **核心职责**：全链路任务编排、论证树状态管理、消息路由、HITL节点触发、异常兜底、任务生命周期管控
- **输入**：上游系统传入的原始文档 + 任务配置（开启/关闭任务、修订等级、是否启用人工确认）
- **输出**：最终修订结果 + 全链路执行日志
- **核心逻辑**：
  1.  接收任务后按配置拆解执行步骤，依次调度下游智能体
  2.  维护论证树全生命周期状态，同步所有模块的修改结果
  3.  在论证树生成后、修订结果输出前触发对应HITL节点，等待人工反馈后更新状态
  4.  接收门禁打回结果，定向重调度对应智能体重执行
  5.  单智能体执行异常时触发降级兜底策略，保障流程不中断

### 3.2 主干层：论证结构解析Agent
- **核心职责**：全文唯一语义解析入口，输出标准化论证树，作为全链路唯一数据主干
- **输入**：预处理后的纯文本 + 段落映射关系
- **输出**：标准化论证树（节点集合 + 层级关系 + 元数据）
- **核心逻辑**：
  1.  全文语义分区：将内容划分为事实型论据、观点型论断、限定条件、背景叙述、主观评价5类，仅前3类纳入论证节点
  2.  层级构建：自顶向下识别「主论点→分论点→论据→限定条件」的从属关系，构建树状结构
  3.  元数据标注：为每个节点绑定段落ID、原文位置、论证权重、适用边界
  4.  状态初始化：所有原始节点状态标记为「未校验」

### 3.3 任务执行层（双任务并行、可插拔）
#### 3.3.1 假设生成Agent
- **核心职责**：基于论证树目标节点的完整上下文，生成符合原文论证框架的可验证假设
- **输入**：目标论点节点ID + 完整论证树
- **输出**：假设列表（含假设内容、挂载节点ID、假设类型、初始置信度），自动挂载至对应论点节点下
- **核心逻辑**：
  1.  直接从论证树拉取目标节点及其下属所有论据、限定条件的完整上下文，无需额外定位与聚合
  2.  严格在原文适用边界内生成可证伪的假设，禁止无依据外推
  3.  生成后自动调用公共检索层获取支撑/反驳材料，补充假设的验证依据与支撑度
  4.  假设节点状态标记为「待验证」，关联检索到的来源素材

#### 3.3.2 事实验证Agent
- **核心职责**：基于论证树的论断-论据从属关系，调用多源检索校验论据真实性，输出校验结论
- **输入**：目标论据节点ID + 完整论证树
- **输出**：论据校验结果（可信/存疑/错误）、错误类型、修正建议、对应参考来源
- **核心逻辑**：
  1.  直接复用论证树的父子节点关系，提取「论断-论据」配对，无需二次抽取
  2.  调用公共检索层获取权威对照素材，完成语义比对与事实校验
  3.  数值类论据专项校验数值准确性与统计口径一致性
  4.  无权威来源的主观表述标记为「存疑」，不做强制判定

#### 3.3.3 影响传导Agent
- **核心职责**：论据/假设的校验结果沿论证树向上传导，评估对上层论点的影响，输出修订决策
- **输入**：节点校验结果 + 论证树层级关系
- **输出**：论点影响评估（支撑充足/支撑不足/论点失效）+ 修订建议 + 完整影响链路
- **核心逻辑**：
  1.  按论证权重计算单条论据/假设失效后的剩余支撑力度
  2.  支撑力度低于阈值时，向上逐层传递影响直至主论点
  3.  输出对应修订方案（修正论据/弱化论点/补充论据），避免孤立修改导致逻辑断裂

### 3.4 公共检索能力层（底层公共支撑、全链路复用）
三类检索Agent统一接收检索请求，返回带来源标识的素材结果，同时服务于事实验证与假设验证。

#### 3.4.1 网络检索Agent
- **核心职责**：白名单范围内的公开网络信息检索，获取官方政策、公开标准、行业公开数据等素材
- **输入**：检索关键词 + 检索类型（政策/数据/案例）
- **输出**：检索结果列表（含内容片段、来源URL、发布机构、发布时间）
- **核心约束**：仅访问预置官方白名单域名，禁止泛网搜索；检索请求脱敏，不传入文档敏感内容

#### 3.4.2 知识库检索Agent
- **核心职责**：内部非结构化知识库的语义检索，获取历史方案、内部规范、沉淀案例等素材
- **输入**：检索向量 + 权限标识
- **输出**：召回的知识库片段（含内容、文档名称、权限等级）
- **核心约束**：调用前自动校验用户权限，越权内容不召回、不返回；支持按文档类型、时间范围过滤

#### 3.4.3 结构化数据检索Agent
- **核心职责**：内部关系型数据库的精准查询，获取量化指标、业务数据、实体属性等结构化素材
- **输入**：查询条件 + 数据维度
- **输出**：结构化数据结果（含数值、统计口径、数据时间）
- **核心约束**：仅执行预定义查询模板，禁止泛化SQL查询；严格遵循数据权限隔离规则

### 3.5 门禁层：一致性校验Agent
- **核心职责**：两级质量校验，独立于所有生成类智能体，仅做验收不做修改
- **输入**：更新后的完整论证树
- **输出**：校验通过/打回 + 问题节点定位 + 问题说明
- **核心逻辑**：
  1.  段落级校验：单段内论点与论据自洽性、假设与原文逻辑边界匹配度
  2.  全局校验：跨段落论点一致性、论据口径统一性、术语定义一致性
  3.  打回后由调度Agent定向重调度对应智能体，直至通过或触发人工干预阈值

### 3.6 输出层：修订回写Agent
- **核心职责**：将结构化论证树的所有变更，转换为自然语言修订版文档
- **输入**：最终状态论证树 + 原文位置映射关系
- **输出**：修订版全文 + 修订痕迹清单
- **核心逻辑**：
  1.  仅替换命中问题的局部内容，保留原文整体行文逻辑与表达风格
  2.  新增假设与补充论据标注专属标识，附在对应论点所在段落末尾
  3.  生成修订明细清单，包含修改位置、修改原因、对应节点ID、参考来源

## 4. HITL（人在回路）节点定义
共两个核心人工确认节点，均由全局调度Agent触发，人工操作结果直接更新论证树状态。

### 4.1 节点1：论证结构确认（切分确认）
- **触发时机**：论证结构解析Agent输出论证树后，任务执行层启动前
- **输入**：初始论证树 + 原文映射关系
- **人工操作**：调整节点层级、合并/拆分节点、修正节点边界、标记无需处理的段落
- **输出**：人工确认后的最终版论证树
- **规则**：支持跳过，跳过后直接使用初始论证树进入下一环节

### 4.2 节点2：修订内容确认
- **触发时机**：双任务执行完成、一致性校验通过后，最终回写前
- **输入**：带修订建议的论证树 + 修订明细
- **人工操作**：单条采纳/驳回修订建议、手动修改节点内容、新增/删除假设节点
- **输出**：人工确认后的终版论证树
- **规则**：支持按段落粒度确认，支持批量确认低风险修订项；驳回项可附带意见触发重生成

## 5. 全链路编排流程
### 5.1 全量修订完整协作流程
1.  全局调度Agent接收原始文档与任务配置，初始化任务
2.  调度论证结构解析Agent，生成初始论证树
3.  触发HITL节点1：论证结构确认，人工确认后固化论证树主干
4.  并行调度双任务执行：
    - 假设生成链路：假设生成Agent生成待验证假设 → 调用公共检索层获取支撑材料 → 更新假设节点状态
    - 事实验证链路：事实验证Agent调取论据节点 → 调用公共检索层完成校验 → 影响传导Agent向上传导结果 → 更新节点状态
5.  调度一致性校验Agent执行全局校验
6.  校验不通过则定向重调度对应智能体，校验通过则触发HITL节点2：修订内容确认
7.  人工确认完成后，调度修订回写Agent生成最终修订文档
8.  全局调度Agent聚合最终结果与全链路日志，返回上游系统

### 5.2 单任务独立调用流程
- **仅假设生成**：跳过事实验证与影响传导，完成假设生成与检索验证后，直接输出假设清单与挂载后的论证树
- **仅事实验证**：跳过假设生成，完成论据校验与影响传导后，直接输出校验结果与影响评估报告

## 6. 统一数据标准与协作协议
### 6.1 论证树节点数据规范（全链路唯一数据标准）
| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| node_id | string | 是 | 节点唯一标识，全链路通用 |
| node_type | enum | 是 | 主论点/分论点/论据/限定条件/假设 |
| parent_id | string | 否 | 父节点ID，顶层节点为null |
| children_ids | array | 是 | 子节点ID列表 |
| paragraph_id | string | 是 | 所属段落ID |
| text_span | object | 是 | 对应原文起止偏移量 |
| content | string | 是 | 节点对应原文/生成/修订内容 |
| argument_weight | int | 是 | 论证权重0-100，用于影响传导计算 |
| boundary | string | 否 | 适用边界与限定条件摘要 |
| status | enum | 是 | 未校验/待验证/可信/存疑/错误/已修正/失效 |
| evidence_list | array | 否 | 支撑/反驳素材列表，含来源、内容片段 |
| support_score | int | 否 | 支撑度得分0-100 |

### 6.2 智能体间消息协议
所有交互通过全局调度Agent转发，采用标准化异步消息格式：
- 消息头：任务ID、智能体ID、消息类型（任务/结果/异常/HITL）
- 消息体：业务数据、对应节点ID列表、执行状态
- 支持超时重试、失败降级、状态幂等校验

### 6.3 节点状态机
所有节点状态流转严格遵循规则，非法变更直接拦截：
- 初始状态：未校验
- 假设生成后：待验证
- 检索校验后：可信/存疑/错误
- 人工/系统修订后：已修正
- 论据失效后：失效

## 7. 核心协作规则与硬约束
1.  **唯一主干规则**：所有智能体仅从论证树获取与写入数据，禁止私存、私改原始文本，所有变更均留痕可追溯。
2.  **检索公共复用规则**：所有事实类校验统一调用公共检索层，禁止各任务私接数据源，保障来源口径统一、权限统一。
3.  **无依据不输出规则**：所有事实论断、生成假设必须绑定至少1个有效检索来源，无来源内容标记为存疑，不强制输出结论。
4.  **任务解耦规则**：两大任务无直接依赖，可独立启停、独立迭代，一方异常不影响另一方执行。
5.  **门禁独立规则**：一致性校验Agent不参与任何生成逻辑，仅做中立验收，保障质量判断客观性。
6.  **异常兜底规则**：单智能体执行失败3次后触发降级，标记对应节点为待人工处理，保障整体流程不中断。

## 8. 非功能指标
### 8.1 效果指标
- 论证结构识别准确率：≥90%（论点-论据从属关系正确率）
- 事实验证准确率：≥85%
- 假设有效率：≥80%（符合原文逻辑边界、具备验证价值的假设占比）
- 检索素材召回准确率：≥90%
- 全局矛盾漏检率：≤5%

### 8.2 性能指标
- 1万字文档：论证树解析≤30s，全量双任务执行≤120s
- 单节点检索校验：平均耗时≤2s
- 系统并发支持：≥50个并行任务

### 8.3 可靠性指标
- 单智能体异常不导致全链路失败，任务成功率≥99%
- 全链路执行日志完整留存，支持节点级追溯审计
- 支持私有化部署，数据全程不出域
