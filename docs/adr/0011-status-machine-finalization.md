# ADR-0011：状态机收尾——新增 adopted 中间态，corrected 由回写成功触发

## 状态

已接受（2026-07-10）

## 背景

PRD 3.2 列了终态 `corrected`（已修正）与 `invalid`（失效），但未定义触发点：
- `corrected` 说「人工/系统修订后」，未区分「HITL-2 勾选」还是「回写成功」两个时点。
- `invalid` 说「论据失效后」，未定判定者，与 `error` 界限不清。

若勾选即置 `corrected`，而回写缝合失败，则状态机撒谎（显示已修正、原文未变），且用户决策在重试失败时丢失。

## 决策

1. **新增中间态 `adopted`（已采纳·待回写）**：
   - HITL-2 用户采纳假说 → 节点进 `adopted`，**同时持久化「采纳了哪条假说」**（回写的幂等依据）。
   - 这是决策的持久记录，防止回写失败 + 重试失败时丢失用户决定。
2. **`corrected` 由回写成功触发**，不在勾选时打（决策与既成事实分离）。
3. **回写失败 → 节点停留 `adopted`**（不置 corrected），贴错误标签。重试扫描「`adopted` 且未 `corrected`」节点，读其已采纳假说重做回写，**幂等可恢复**。
4. **`error` vs `invalid` 分工**：
   - `error` = 事实验证对**叶子论据**的判决（论据自证其伪）。
   - `invalid` = 影响传导对**上层论点**的判决（支撑塌陷、剩余支撑力度低于阈值而失效）。

## 状态机全貌

```
unverified → pending_verification → (credible | doubtful | error)
                                         ↓ HITL-2 采纳
                                     adopted ──回写成功──→ corrected
                                        └──回写失败──→ 停留 adopted（可重试）
invalid：由影响传导对上层论点单独判定
```

## 影响

- 节点状态枚举新增 `adopted`。
- 节点需持久化「已采纳假说」引用。
- 回写是幂等操作，可安全重试。
