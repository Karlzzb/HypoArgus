# ADR-0003：影响传导为 Merge 之后的串行阶段，且不产文本

## 状态

已接受（2026-07-10）

## 背景

PRD 拓扑图将「假设生成 / 事实验证 / 影响传导」三者并列于「双任务执行层（并行）」。
但影响传导的逻辑依赖「单条论据失效后的剩余支撑力度」，其输入必须是事实验证已出结论之后才存在——物理上不可能与事实验证并行。
此外影响传导也「输出修订建议」，与假设生成的 `candidate_hypotheses` 构成两个修订来源，可能落到同一上层节点，造成 HITL-2 呈现混乱。

## 决策

将流水线明确为两拍：

```
第一拍（真并行）：事实验证 ∥ 假设生成（投机）
      ↓  双轨合并算子
第二拍（串行）：  影响传导 → 一致性校验 → HITL-2
```

1. **影响传导挪到 Merge 之后的串行第二拍**，读取合并后的树。
2. **影响传导不产任何替代文本**，只做两件事：
   - 将失效沿论证权重向上传导，为上层节点打状态 / 权重衰减；
   - 若某上层 `sub_claim` 因此塌方，**触发复用该节点已有的投机假设**去激活，而非另开一路处方。
3. **修订假设的唯一来源是假设生成 Agent**。影响传导只决定「哪些上层节点也该亮灯」。

## 权衡

- 牺牲拓扑图「三 Agent 并行」的对称美感，换取因果自洽与 HITL-2 呈现的单一来源。
- 影响传导不产文本，避免了两套处方在同一节点冲突。

## 影响

- 拓扑图需修正：影响传导不属于并行层。
- HITL-2 中「候选修订假设」永远只源自假设生成 Agent。
- 上层 `main_claim` / `sub_claim` 的候选假设可能由「投机生成」或「传导触发激活」两条路点亮，但文本来源同一。
