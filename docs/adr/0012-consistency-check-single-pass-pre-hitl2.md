# ADR-0012：一致性校验只在 HITL-2 前扫一次，采纳后不回炉

## 状态

已接受（2026-07-10）

## 背景

需澄清一致性校验的时机与扫描对象。此前用「合并后的树」表述有歧义——系统全程只有**一棵树**（唯一数据主干），「双轨合并算子」合并的是两条线路的**结果标注**，不是合并两棵树。

## 决策

1. **顺序**（承 ADR-0003）：Merge → 影响传导 → **一致性校验** → HITL-2。
2. **扫描对象**：那棵已带好 `status` + 激活假说 + 传导结果的树。
3. **只扫一次**，产出 `issue_tags` 喂给 HITL-2。
4. **看不到 `adopted / corrected`**（HITL-2 之后才有）。**不因用户采纳可能引入新冲突而回炉重扫**——遵循「单向不打回、无 Rejection Loop」原则。

## 权衡

- 已知缺口：用户在 HITL-2 采纳的多条修订之间若互相打架，本版**不做二次自动检验**，靠用户在界面自行审视。
- 换取流水线单向一冲到底的稳健性，避免打回死循环。后续版本可考虑「采纳后轻量二次校验」。

## 影响

- 一致性校验是无状态的一次性扫描，不参与 HITL-2 之后的流程。
