# ADR-0001：以段落为回写原子单位，回写不依赖绝对偏移量

## 状态

已接受（2026-07-10）

## 背景

论证节点数据同时携带 `text_span`（全文绝对偏移量）与 `paragraph_id`（段落映射）。
回写 Agent 需要「只重写被改动的局部、其余原样复制」，且测试要求未改动内容达到字节级对齐。
两套定位信息并存产生冗余与张力：绝对偏移量在前序段落长度变化后会全局失效，维护成本高且脆弱。

同时需要厘清映射基数：一个段落能否含多个节点、一个节点能否跨段落。

## 决策

1. **段落（`paragraph_id`）是回写的唯一原子单位。** 回写逻辑完全不依赖 `text_span` 绝对偏移量。
2. `text_span` 降级为**段内辅助定位信息**，不参与回写缝合。
3. **基数约束**：一个段落可包含多个节点；**一个节点不可跨段落**。解析发现跨段结构时，须在 HITL-1 由用户拆分，或整块并为单一节点。
4. **字节级对齐的粒度重定义**：保证「不含任何被勾选改动节点的段落」字节级无损；一旦段内有节点被改，整段进入重写通道，段内其他句子不作硬字节承诺（重写 Agent 应尽量保留）。

## 权衡

- 放弃了节点级细粒度缝合，换取回写的强确定性与可测试性。
- 代价：含多节点的段落中，若一个节点被改，同段其他句子失去字节级保护。可接受，因专业文档段落通常语义内聚。

## 影响

- 回写 Agent 不维护、不信任全局偏移量。
- 测试用例「非变更段落字节级还原」以段落为断言粒度。
